### **Решение**
Архитектурное решение основано на расширении существующей депозитной платформы новыми кредитными сервисами с максимальным переиспользованием готовой инфраструктуры.

**Ключевые архитектурные решения:**

1. **Расширение существующих компонентов**:
   - **API Gateway** - добавление маршрутов для кредитных API
   - **Auth Service** - переиспользование без изменений
   - **Notification Service** - расширение для кредитных уведомлений
   - **Event Bus (Kafka)** - добавление новых топиков для кредитных событий
   - **SFTP File Server** - расширение для передачи кредитных ставок партнерскому кол-центру
   - **Call Center System** - дополнение интерфейса кредитными продуктами через существующую интеграцию

2. **Новые специализированные сервисы**:
   - **Credits Service** - управление кредитными заявками и предодобренными предложениями
   - **Scoring Integration Service** - адаптер для интеграции с существующей Scoring System
   - **BKI Adapter** - сервис для работы с Бюро кредитных историй
   - **Credit Conveyor Adapter** - интеграция с существующим Credit Conveyor

3. **Существующие системы (без изменений)**:
   - **Credit Conveyor (Camunda)** - остается основной системой обработки кредитов
   - **Scoring System (Python/Flask)** - сохраняет текущую логику скоринга
   - **АБС** - интеграция через существующий ABS Adapter

4. **Новые интеграции**:
   - Прямая интеграция с БКИ через REST API
   - Асинхронная связь между новыми и существующими системами через Kafka

**Технологический стек (дополнения к существующему):**
- Credits Service: Java Spring Boot (консистентность с существующими сервисами)
- Scoring Integration Service: Java Spring Boot
- BKI Adapter: Java Spring Boot
- Credit Conveyor Adapter: Java Spring Boot
- База данных: PostgreSQL (для новых кредитных данных)
- Кэш: Redis (для предодобренных предложений)

**Преимущества расширения существующей архитектуры:**
- Переиспользование 80% инфраструктуры (API Gateway, Auth, Notification, Event Bus, SFTP, Call Center integration)
- Минимальные изменения в legacy-системах
- Быстрое время внедрения за счет готовой кол-центр интеграции
- Консистентный пользовательский опыт для депозитов и кредитов
- Единая поддержка операторов кол-центра по всем банковским продуктам

### **Альтернативы**

**Альтернатива 1: Использование только существующего Credit Conveyor**
- Добавление веб-интерфейса напрямую к Credit Conveyor
- Преимущества: Минимальные изменения архитектуры
- Недостатки: Credit Conveyor не предназначен для высоконагруженных веб-запросов, отсутствие предварительной оценки

**Альтернатива 2: Создание отдельной кредитной платформы**
- Полностью независимая система кредитования
- Преимущества: Полная свобода в архитектурных решениях
- Недостатки: Дублирование инфраструктуры, сложность интеграции, увеличение операционных затрат

**Альтернатива 3: Интеграция через существующую Scoring System**
- Использование Scoring System как основной точки входа для кредитов
- Преимущества: Централизованная логика оценки
- Недостатки: Scoring System не предназначена для управления заявками, отсутствие workflow-возможностей

**Недостатки, ограничения, риски**

**Недостатки выбранного решения:**
1. Увеличение сложности системы за счет дополнительных сервисов-адаптеров
2. Зависимость от стабильности существующих систем (Credit Conveyor, Scoring)
3. Необходимость синхронизации между новыми и legacy-системами

**Ограничения:**
1. Credit Conveyor остается узким местом для обработки сложных кредитных заявок
2. Пакетная передача данных между АБС и Credit Conveyor (раз в сутки) ограничивает real-time обновления
3. Ограниченные возможности кастомизации Scoring System для новых требований

**Риски:**
1. **Интеграционный риск**: Сложность интеграции с БКИ и обеспечения стабильности
2. **Производительный риск**: Нагрузка на Scoring System при увеличении количества заявок
3. **Операционный риск**: Усложнение мониторинга и поддержки за счет увеличения количества компонентов
4. **Бизнес-риск**: Возможные задержки в обработке кредитов при сбоях в legacy-системах

**Митигации рисков:**
- Поэтапное внедрение с тщательным нагрузочным тестированием
- Реализация circuit breaker pattern для всех внешних интеграций
- Расширенный мониторинг производительности Scoring System
- Кэширование результатов скоринга для повторных запросов
- Резервные механизмы обработки при недоступности БКИ
- Детальное логирование всех кредитных операций для аудита
